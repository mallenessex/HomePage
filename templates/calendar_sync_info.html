Sync with {{ provider }} Calendar

To enable syncing with your {{ provider }} calendar, you will need to provide API credentials. This ensures your data remains under your control without relying on a central HOMEPAGE gateway.

⚠ External Internet Required: Calendar sync connects to {% if provider == 'Google' %}Google{% else %}Microsoft{% endif %} servers. This feature should only be configured by an administrator.

Setup Instructions:

    {% if provider == 'Google' %}
  1. Go to the Google Cloud Console (console.cloud.google.com).
  2. Create a new project (e.g., "Family Homepage").
  3. Enable the Google Calendar API.
  4. Configure the OAuth consent screen.
  5. Create OAuth 2.0 Client ID credentials for a "Web Application".
  6. Add this Authorized Redirect URI:
    {{ request.base_url }}calendar/sync/google/callback
    7. {% else %}
  7. Go to the Azure Portal — App Registrations (portal.azure.com).
  8. Register a new application.
  9. Under Authentication, add a Web platform.
  10. Add this Redirect URI:
    {{ request.base_url }}calendar/sync/outlook/callback
  11. Under API Permissions, add Calendars.ReadWrite for Microsoft Graph.
    12. {% endif %}

After creating your credentials, add them to your .env file in the V2 directory: 

{% if provider == 'Google' %}GOOGLE_CLIENT_ID="your_client_id"
GOOGLE_CLIENT_SECRET="your_client_secret"
{% else %}MICROSOFT_CLIENT_ID="your_client_id"
MICROSOFT_CLIENT_SECRET="your_client_secret"
{% endif %}

Once added, restart the server and the sync buttons will become active!
